Room Temperature Display with Alerts

A portable device that measures ambient temperature and displays it on an LCD, with LED/buzzer alerts when the value is outside user-set bounds. I designed the interface circuitry, wrote the Arduino firmware, and modeled a simple enclosure in Onshape. Tinkercad was used to draft and verify the circuit connections before soldering.

Goals & Requirements

  • Measure ambient temperature and display in °C/°F (toggle).
  • Set min/max thresholds; alert with LED + buzzer when out of range.
  • Stable, readable output (simple filtering) and quick threshold UI.
  • Portable enclosure; battery or USB power.

System Architecture

The sensor’s analog voltage is sampled by the Arduino ADC, converted to temperature, and shown on a 16×2 LCD over I²C. A small UI lets you adjust threshold bounds; when the temperature crosses them, the device drives an LED and piezo buzzer. Circuit planning was done in Tinkercad, then transferred to a soldered prototype and enclosed in a printed case.

Firmware & Code

The firmware samples the sensor, applies a moving average, updates the LCD, and checks thresholds with hysteresis to avoid chatter. Buttons (or a rotary encoder) adjust the min/max setpoints; the LED and buzzer indicate out-of-range conditions. Paste your full Arduino sketch below so reviewers can read the actual implementation.

void setup()
  {
    Serial.begin(9600); //Start the serial connection with the computer
    pinMode(buzzer,OUTPUT); //sets buzzer pin as an output pin
    pinMode(redLED, OUTPUT); //sets redLED pin as an output pin

void loop()
  {
    {
      int reading = analogRead(sensorPin); //getting the voltage reading from the temperature sensor
      float voltage = (reading * 5.0)/1024; // converting that reading to voltage
      float temperatureC = (voltage - 0.5) * 100 ; //converting from 10000 mv per degree with 500 mV offset to degrees
      float temperatureF = (temperatureC * 9.0 / 5.0) + 32.0; // converting to Fahrenheit
      lcd.clear(); // Clears display from previous loop
      lcd.setCursor(0,0);
      lcd.print("Temperature(F): "); // Print characters to LCD.
      lcd.setCursor(0,1);
      lcd.print(temperatureF); // Print variable value to LCD

      if (temperatureF < 75 || temperatureF > 80) // if the temperature is between 70-80 degrees F, the if statement will run
      {
         //redLED code
         digitalWrite(redLED, HIGH); // turns on red LED
         delay(100); //red LED reamins on for 0.1 sec
         digitalWrite(redLED, LOW); // turns off red LED
         delay(100); //red LED reamins off for 0.1 sec

         //buzzer code
         tone(buzzer, 1000); // buzzer will play a 1KHz tone
         delay(100); // buzzer will ring for 0.1 sec
         noTone(buzzer); // stops the buzzers ring
         delay(1000); // pauses the ring for 0.1s
      }
      else // if the temperature is not between 70-80 degrees F, the else statement will run
      {
         digitalWrite(redLED, LOW); // sets red LED to off
         digitalWrite(buzzer, LOW); // sets buzzer to off
      }
      delay(1000); // the loop will wait 1 sec before it re-loops
    }
  }

// template code was taken from https://www.princetronics.com/supermariothemesong/ and modifed to fit the rubric of the project.
void buzz(int targetPin, long frequency, long length)
  {
    digitalWrite(2, HIGH);
    long delayValue = 1000000 / frequency / 2; // calculate the delay value between transitions
    long numCycles = frequency * length / 1000; // calculate the number of cycles for proper timing
    for (long i = 0; i < numCycles; i++) // for the calculated length of time...
    {
      digitalWrite(targetPin, HIGH); // write the buzzer pin high to push out the diaphram
      delayMicroseconds(delayValue); // wait for the calculated delay value
      digitalWrite(targetPin, LOW); // write the buzzer pin low to pull back the diaphram
      delayMicroseconds(delayValue); // wait again or the calculated delay value
    }
    digitalWrite(2, LOW);
  }

Implementation Notes

  • Sensor: TMP36 is simplest; thermistor requires divider and β or Steinhart–Hart constants.
  • Filtering: 8–16-sample moving average smooths noise with minimal lag.
  • UI: Buttons/encoder to select min/max and adjust values; optional °C/°F toggle.
  • Alerting: LED steady on; buzzer short chirps every N seconds while out of range; add ±0.3 °C hysteresis.
  • Power: 9 V battery to VIN (onboard 5 V regulation) or USB; choose LED resistors to limit current.
  • Enclosure: Onshape model with standoffs, ventilation slots, and access to UI/USB.
